A post in R Markdown

by Danielle Navarro, 13 Apr 2019



Slum provides a simple Hugo theme intended primarily to be used in conjunction with the blogdown package. Posts take the form of R Markdown documents. Markdown is a simple formatting syntax for authoring HTML, PDF, and MS Word documents. For more details on using R Markdown see http://rmarkdown.rstudio.com. You can embed code chunks like this:

library(tidyverse)
library(slumstyle)

The theme is designed on the assumption that you’re writing pages in R Markdown rather than plain markdown. I’m pretty sure it supports plain markdown (I didn’t do anything to break it, anyway), but I’ve been writing with an assumption that eventually the colour schemes used by the Hugo template will be mirrored in some simple ggplot themes. For instance, this page uses the default “dark” colour scheme. To generate plots that integrate nicely with this scheme, currently you have to do something like Figure 1:

ggplot(mpg, aes(displ, hwy)) + 
  geom_point() + theme_slum("dark")

Figure 1: Here is a plot

The theme_slum() function here is provided by the slumstyle package, and is useful for styling plots in a fashion that is consistent with the slum style(s). By default the slum theme provides this “dark” colour scheme, but there is also a “light” scheme and a “kunoichi” scheme. The default colour scheme is set in the config.toml file and specifies the path to a css file. It’s not too difficult to create a new colour scheme. The static/css/colour_dark.css file specifies the four main colours used:

:root {
  --primary: #ddd;
  --subtle: #888;
  --behind: #333;
  --highlight: hotpink;
}

It is possible to have different posts using different colour schemes (see “A light themed post”). At the moment, blockquotes look like this:

One is the loneliest number that you ever knew
Two can be as bad as one, it’s the loneliest number since the number one
